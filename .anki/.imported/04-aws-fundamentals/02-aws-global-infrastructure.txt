"What is an AWS Region?","An isolated geographic area with complete AWS infrastructure (compute, storage, database, etc.). Data stays within a region unless explicitly replicated.","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
"What is an Availability Zone (AZ)?","An isolated infrastructure cluster within a region, each capable of independent failure.","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
"What are the key benefits of AWS Regions?","<b>Fault isolation</b> (regional failures don't affect other regions), <b>Compliance</b> (resources stay within country/laws), and <b>Performance</b> (lower latency for nearby users).","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
"Why are applications deployed across multiple AZs?","To remain available during single-AZ outages, providing resilience.","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
"What are Edge Locations?","Small data centers distributed globally, primarily used for content delivery (CloudFront), caching, and edge computing.","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
"How is the scale of Edge Locations compared to Regions?","There are more Edge Locations than Regions, providing better performance for end users worldwide.","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
"What services have Global Resilience?","Services like <b>Route 53</b> that remain available unless <b>ALL regions fail</b>.","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
"What services have Region Resilience?","Services like <b>S3</b> that remain available unless <b>ALL AZs in a region fail</b>.","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
"What services have Availability Zone Resilience?","Services like a single-AZ <b>EC2 instance</b> that are only available if that specific AZ is available.","SAA-C03::04 AWS Fundamentals::02 AWS Global Infrastructure"
