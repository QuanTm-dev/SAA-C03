# Elastic Kubernetes Service (EKS) 101

> EKS is AWS's implementation of K8S.

## Deployment Options

EKS can run on:

- **AWS Cloud**
- **AWS Outposts**: AWS-managed infrastructure on-premises
- **EKS Hybrid Nodes**: AWS-Managed control plane with customer-managed on-premises/edge nodes
- **EKS Anywhere**: Run EKS on premises.
- **EKS Distro**: Open source EKS.

## Control Plane Architecture

- Runs across multiple AZ
- etcd distributed across multiple AZs

## Node Deployment Options

- **Self-Managed Nodes**: Run on EC2 instances you provision and manage.
- **Managed Node Groups**: Run on EC2 instances AWS manages for lifecycle and provisioning. You're responsible for choosing the server types and when to scale.
- **Fargate**: Nodes run on AWS-managed shared infrastructure. You don't need to choose server type or decide when to scale.
- **EKS Auto Mode**: Function like Fargate, but use EC2 instances. A replacement of Fargate.

## Control Plane to Node Communication

- Uses Kubernetes API server endpoint (kube-api) via private ENI or public endpoint.

## AWS Integration & Storage

- Integrates with CloudWatch, IAM, Auto Scaling Groups, NLB/ALB
- Supports AWS storage services: EBS, EFS, S3, FSx, S3 Files
