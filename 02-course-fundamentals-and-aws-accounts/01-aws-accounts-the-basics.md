- AWS account is a container for your AWS resources and the identities (users) created inside it.

- Each AWS account has a root user, created with the email used to sign up, and it has full access to all services and resources in the account.

- Each AWS account has a payment method, which is the credit card that you used to sign up. The payment method is used to pay for the usage of any AWS Services you consume in that account.

- Inside an AWS account, you can create multiple identities (users, groups, roles) using AWS Identity and Access Management (IAM). These identities start off with no permissions, and you can assign them permissions to allow them to access specific AWS services and resources inside that account.

- By default, external identities (users, groups, roles) created in one AWS account cannot access resources in another AWS account. However, you can set up cross-account access using IAM roles.
