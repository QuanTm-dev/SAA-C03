# CloudWatch (CW)

> Core AWS monitoring and management service providing real-time insights into resource performance, logs, and system events.

## Core Components

- **Metrics**: Numerical data points representing resource/application performance over time.
- **Logs**: Log data from AWS services, applications, and custom sources.
- **Alarms**: Triggered when metrics exceed/fall below thresholds; can send notifications or trigger actions.
- **Events**: JSON documents describing AWS resource changes.

## Key Terms

- **Namespace**: Container for metrics; each AWS service has its own namespace.
- **Metric**: Time-ordered set of data points for a resource/application.
- **Data Point**: Single measurement at a specific time.
- **Dimension**: Name-value pair that uniquely identifies a metric.
- **Alarm**: Tool that monitors a metric against a threshold; performs actions (notifications, auto-scaling, etc.) when threshold is crossed.
