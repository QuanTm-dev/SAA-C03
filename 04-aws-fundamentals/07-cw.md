# Basics

- CloudWatch (CW) is a core supporting service within AWS which provides metric, log and event management services.

- CW contains the following key components:
  - **Metrics**: Numerical data points that represent the performance of AWS resources and applications over time.
  - **Logs**: Log data generated by AWS services, applications, and custom sources.
  - **Alarms**: Notifications triggered based on specified thresholds for metrics.
  - **Events**: Real-time stream of system events that describe changes in AWS resources.

# Terms

- **Namespace**: A container for CloudWatch metrics. Each AWS service has its own namespace.
- **Metric**: A time-ordered set of data points that represent the performance of a resource or application.
- **Data Point**: A single measurement of a metric at a specific time.
- **Dimension**: A name-value pair that helps to uniquely identify a metric.
- **Alarm**: A mechanism that monitors a single metric over a specified time period and performs actions based on the value of the metric relative to a threshold. E.g., sending notifications or triggering auto-scaling.
