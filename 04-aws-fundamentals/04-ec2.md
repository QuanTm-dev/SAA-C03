# EC2: Elastic Compute Cloud

> IaaS service providing virtual machines (instances) with configurable CPU, memory, storage, and network resources.

## Basics

- Instances deploy within a VPC to a specific Availability Zone.
- **Resilience**: AZ-resilient (instance fails if its AZ fails).
- Storage options: EBS volumes (persistent) or instance store (temporary).
- Multiple instance types available with varying capabilities (CPU, memory, GPU, etc.).

## Billing

- **Running**: Charged per second for instance, EBS storage, and network transfers.
- **Stopped**: Charged only for attached EBS storage.
- **Terminated**: No charges; data is lost permanently.

## Instance Lifecycle

| State          | Description                                                               |
| -------------- | ------------------------------------------------------------------------- |
| **Running**    | Instance is active and consuming resources.                               |
| **Stopped**    | Instance is offline; EBS volumes are retained. Restart with no data loss. |
| **Terminated** | Instance is deleted permanently; data lost. Cannot be reversed.           |

## Amazon Machine Image (AMI)

**Definition**: Template containing OS, filesystem, and pre-installed software. Used to create EC2 instances.

**Permissions**

- **Public**: Anyone can launch instances.
- **Private**: Only the owner can launch instances.
- **Shared**: Specific AWS accounts can launch instances.

**Boot Volume**: Root EBS volume containing the OS and filesystem (e.g., C: drive in Windows, / in Linux).

**Block Device Mapping**: Defines all attached storage volumes and identifies the boot volume. Tells the OS how to recognize and mount devices.
