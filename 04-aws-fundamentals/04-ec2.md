# EC2 Basics

**What is EC2?**
Elastic Compute Cloud (EC2) is an IaaS service providing virtual machines called instances. Instances are OS with allocated CPU, memory, storage, and network resources.

**Architecture & Resilience**
EC2 instances deploy within a VPC to a specific Availability Zone. EC2 is AZ-resilient (instance fails if its AZ fails), and can use EBS volumes for persistent storage or instance store for temporary local storage.

**Instance Types & Billing**
EC2 offers various instance types with different capabilities (CPU, memory, GPU, etc.). Billing is per-second while the instance runs; stopped instances only charge for EBS storage (if attached).

# EC2 Instance Lifecycle

| State          | Charges                                      |
| -------------- | -------------------------------------------- |
| **Running**    | Instance, EBS storage, network data transfer |
| **Stopped**    | EBS storage only (if attached)               |
| **Terminated** | No charges (data lost)                       |

**State Transitions**
Running instances can be stopped and restarted without data loss (EBS volumes persist). Stopped instances can resume to Running. Instances can only be terminated once—termination is permanent and cannot be reversed.

# Amazon Machine Image (AMI)

**Definition**
An AMI is a template used to create EC2 instances. AMI can be created from an existing EC2 instance.

**AMI Components**

**Permissions**

- **Public**: Anyone can launch instances from this AMI
- **Private**: Only the owner can launch instances
- **Shared**: Only specific AWS accounts can launch instances

**Boot Volume**
The root EBS volume containing the operating system and a filesystem. This is the volume the instance boots from (C: drive in Windows, root partition in Linux).

**Block Device Mapping**
Defines all storage volumes attached to instances and identifies which volume is the boot/root volume. This tells the OS how to recognize and mount each storage device.
