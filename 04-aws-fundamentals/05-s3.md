# S3: Simple Storage Service

> S3 is a global object storage service with unlimited capacity. Buckets are region-specific (region-resilient); data stays in the chosen region unless explicitly replicated.

## Core Concepts

- **Objects**: Files up to 5TB with a Key (unique name/path) and metadata.
- **Buckets**: Containers for objects with globally unique names.
- **Resilience**: Region-resilient (survives AZ failures in the region).

## Buckets

**Naming Rules**

- 3-63 characters: lowercase, numbers, hyphens, periods only.
- Must start and end with letter or number.
- Cannot be IP address format (e.g., 192.168.1.1).

**Limits**

- Default: 100 buckets per AWS account (can increase to 1000 via support).
- Each bucket holds unlimited objects.
- Names are globally unique across all AWS accounts.

**Folder Structure**: S3 has a flat structure; folders are simulated using prefixes and delimiters in object keys.

## Use Cases

- Large-scale data storage, distribution, backup.
- Input/output for other AWS services.
- Ideal for unstructured data (images, videos, logs, documents).
