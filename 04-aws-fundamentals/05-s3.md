- S3 is a global AWS service, but buckets are region-specific, making S3 region-resilient. Data stays within the region you choose unless explicitly replicated.
- S3 has unlimited storage capacity and allows multiple users to access data concurrently.
- S3 is cost-effective for large-scale data storage, distribution, and archival.
- S3 has two core components: Buckets (containers) and Objects (files).

# Objects

Objects are files stored in S3 with two core components:

- **Key**: The unique name (path) of the object within a bucket.
- **Value**: The file content, up to 5TB in size.
- **Other components**: Version ID, metadata, access control (ACL), and sub-resources.

# Buckets

Buckets are containers for objects with globally unique names across all AWS accounts. Each bucket is stored in a specific region and can hold unlimited objects.

**Account Limits:**

- Default limit: 100 buckets per AWS account (can be increased upto 1000 via AWS Support).

**Bucket Naming Rules:**

- 3-63 characters long, lowercase letters, numbers, hyphens (-), and periods (.) only.
- Must start and end with a letter or number.
- Cannot be formatted as an IP address (e.g., 192.168.1.1).

**Folder Structure:**
S3 uses a flat structure with no actual folders, but you can simulate folders using prefixes and delimiters in object keys.

# Use Cases

- **Best For**: Large-scale data storage, distribution, backup, and as input/output for other AWS services.
- **Object Storage**: S3 is object storage (not file or block storage), ideal for unstructured data like images, videos, logs, and documents.
