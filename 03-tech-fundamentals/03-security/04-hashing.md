## Hashing Basics

- Hashing is a process of taking an algorithm to convert data into a fixed-length unique representation. You cannot reverse a hash to recover the original data (one-way function).
- Deterministic: Same input always produces the same hash; different inputs produce different hashes.

## Collision Weakness

- A **collision** occurs when two different inputs produce the same hash, which compromises integrity. MD5 is vulnerable to collisions and considered cryptographically broken.
