# S3 Events

- S3 Events allow you to trigger actions in response to specific events that occur in your S3 buckets.

## Common S3 Events

- Object Created: Put, Post, Copy, or CompleteMultipartUpload
- Object Deleted: \*(Both Delete and DeleteMarkerCreated), Delete, DeleteMarkerCreated
- Object Restore: Restore, RestoreCompleted
- Object Replication: OperationMissedThreshold, OperationReplicatedAfterThreshold, OperationNotTracked, OperationFailedReplication

## Supported Destinations

- AWS Lambda
- SQS
- SNS

## How S3 Events Work

1. Configure an event notification on an S3 bucket, specifying the event types and destination.
2. When the specified event occurs, S3 sends a notification to the configured destination.

- Note: Events generated by S3 Principal. You need to add permissions to resource policy of the destination to allow S3 to interact with it.

## Limitation

- S3 Events only supports a limited set of event types and destinations.
- Always consider using EventBridge because it provides more flexibility and supports a wider range of events and targets.
